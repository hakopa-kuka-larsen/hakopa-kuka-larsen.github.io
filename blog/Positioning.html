POSITIONING IN CSS

Relative Positioning

I’m relatively positioned!

For our first example the blue box is positioned relative to the reference point (the red dot). If you inspect the blue elements css in the DOM you will see that it has moved 20px from the top, and 20px from the left of the reference point. The reference point being the default location ofthe element if no positioning parameters were given.

Note that it does not move 20px to the left, this initially confused me.


Absolute Positioning

I’m absolutely positioned!

An absolutely positioned element is taken out of the normal document flow, meaning it doesn't leave empty space where it would have been. Its position is based on the nearest ancestor with a positioning context (like relative, absolute, or fixed). If no such ancestor exists, it is positioned relative to the document's initial containingblock.

The final position is set using the top, right, bottom, and left values. When z-index is specified, the element creates a new stacking context. Additionally, its margins don't collapse with the margins of other elements.

Fixed Positioning

Fixed positioning is different in that it is fixed relative to the browser window or view port, it does not move relative to other content or scrolling it is extremely fixed, useful for elements that you want omnipresent and always on screen.

Look to the bottom right hand of the page to view the element (light red) in its fixed position, if you inspect the html in dom you can see that the element technically resides within the DIV, but the fixed position ignores and overides this
